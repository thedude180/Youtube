2. Use Case & Application Description

CreatorOS is a self-hosted, fully autonomous content operations system built to run a YouTube gaming channel end-to-end without manual intervention. The system continuously processes a back-catalog of gaming stream recordings, extracts highlights, packages them as YouTube Shorts and long-form clips, and publishes them on a fixed schedule (3 Shorts/day + 1 long-form/day).

All operations run server-side in production. There is exactly one YouTube channel owner (a single authenticated user) and one linked YouTube channel. This is not a multi-tenant or third-party tool — every API call is made on behalf of that single channel.

Core automated workflows that require API access:

  • Content publishing — uploads 3–4 videos per day to the channel using videos.insert
  • Metadata management — sets titles, descriptions, tags, and category via videos.update and uploads AI-generated thumbnails via thumbnails.set
  • Playlist organization — automatically creates game-specific playlists and assigns each video to the correct playlist (playlists.insert, playlistItems.insert)
  • Catalog sync — hourly delta sync reads the channel's video list to detect new uploads and update performance scores (videos.list, channels.list, playlistItems.list)
  • Pipeline verification — every 30 minutes, a pipeline tracer verifies recently published videos are live and not stuck (videos.list)
  • Live stream copilot — during live streams, reads and responds to chat for audience engagement (liveBroadcasts.list, liveChatMessages.list, liveChatMessages.insert)
  • Community engagement — AI generates replies to viewer comments on published videos (commentThreads.list, commentThreads.insert)

Why 10,000 units is not enough: A single video upload (videos.insert) costs 1,600 units. Publishing 4 videos per day already consumes 6,400 units — leaving only 3,600 units for all metadata updates, thumbnails, playlist management, catalog sync, pipeline verification, and live stream operations. On back-catalog import days (when the system scans hundreds of existing videos for the first time), read operations alone push total consumption well above 10,000 units before any uploads can complete.

5. Impact of Current Quota Limit

  • Uploads fail daily because the 10,000 unit limit is exhausted by catalog sync and metadata operations before videos.insert can run
  • The autonomous publishing schedule (3 Shorts + 1 long-form per day) cannot be maintained — the channel goes days without new content
  • Back-catalog import (scanning the channel's upload history to import existing videos) consumes large amounts of playlistItems.list + videos.list calls, crowding out upload capacity
  • The system has already implemented a circuit breaker that halts all API calls when quota is exceeded, but this means all automation stops until midnight Pacific time every day

6. Policy Compliance & Data Handling

This application is fully compliant with YouTube API Services Terms of Service:
  • Single-user, single-channel: Only one YouTube account is ever authenticated. All API calls are made on behalf of the channel owner (ET Gaming 247). No third-party channels are accessed.
  • No data resale or scraping: API data is used exclusively to operate the owner's channel. No data is sold, shared, or used to train models for third parties.
  • No fake engagement: The system never purchases, simulates, or inflates views, likes, subscribers, or comments. All interactions are genuine automated responses from the channel owner.
  • OAuth token storage: Tokens are stored server-side in a private PostgreSQL database with no public access. Tokens are only used to authenticate calls on behalf of the authenticated channel owner.
  • Token refresh: OAuth refresh tokens are rotated on a 12-hour schedule using Google's token refresh endpoint.
  • User data deletion: The channel owner can revoke OAuth access at any time via Google Account settings, which immediately removes all stored tokens.
  • Rate limiting: The system enforces an internal circuit breaker that halts all API calls the moment quota is exhausted, preventing excess consumption beyond the daily limit.
  • Live chat: Live chat replies are capped at 8 per hour by the live stream copilot. All replies are clearly non-deceptive automated responses from the channel operator.
